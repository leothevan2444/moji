# 连接与入库

这些设置决定 Moji 如何访问依赖服务，以及完成的下载如何对 Stash 可见。每个表单需要独立保存。

## 连接

分别配置 Stash、Jackett 和 qBittorrent 的基础 URL 与凭据，Moji 会据此生成所需 API 地址。最少字段完整时服务为“已配置”；只有最近一次探测成功才是“就绪”。

qBittorrent 默认保存路径、分类和标签会用于没有单独覆盖值的新任务。需要鉴权获取 torrent 文件时，Moji 使用 Jackett 管理界面密码。凭据会返回当前设置界面，因此应限制 Moji 的访问范围。

## 入库

路径映射不操作文件。Moji 从 qBittorrent 内容路径移除 **qB 下载根目录**，再把相对路径拼到 **Stash 媒体库根目录**。只有 qBittorrent 与 Stash 已通过不同前缀访问同一份文件时才应使用。

文件交付会把该相对路径拼到 **Moji 下载根目录**，再复制、移动或链接到 **Moji 媒体库根目录** 下；用于扫描的路径则拼到 **Stash 媒体库根目录**。已有目标不会被覆盖。

每个根目录必须使用字段所标服务的文件系统视角，容器路径必须与对应挂载一致。`复制` 保留源文件，`移动` 可能中断做种，`符号链接` 要求 Moji 始终能访问源路径。
