# 任务与入库

任务是 Moji 针对单个发行内容保存的持久记录，包含所选种子、qBittorrent 状态、交付路径、Stash 扫描任务、错误和时间。删除任务时执行 **设置 > 系统** 中的删除策略。

## 任务生命周期

1. **待选种**：Moji 搜索 Jackett 并排列可下载候选；手动选择 Jackett 结果时跳过自动选择。
2. **下载中**：种子已提交给 qBittorrent，Moji 定时同步名称、进度、状态和内容路径。
3. **待入库**：下载完成，等待计算入库路径。
4. **搬运中**：仅文件交付模式存在；Moji 复制、移动或创建符号链接。
5. **扫描中**：Moji 请求 Stash 扫描计算出的媒体库路径，并跟踪 Stash 后台任务。
6. **已完成**：Stash 报告该扫描任务完成。

每个阶段还可能是待处理、进行中、受阻或已完成。“受阻”会保留可操作错误。重试从当前阶段继续，不会重新下载已经完成的内容。

## 管理任务

搜索范围包括番号、标题、tracker、阶段、种子标识和路径。可筛选运行中、完成、受阻或待扫描，再按创建时间、更新时间或进度排序。**刷新** 重新读取存储状态；**同步进度** 向 qBittorrent 获取当前状态；**触发扫描** 处理符合条件的已完成下载；**重试受阻任务** 批量重试所有受阻项。

自动选种没有可接受结果时，打开 **人工处理** 搜索并指定候选。提交前必须确认候选属于目标番号。删除任务可能同时删除 qBittorrent 任务和文件，具体取决于系统策略；确认窗口会显示当前效果。

## 入库方式

路径映射不操作文件。Moji 从 qBittorrent 内容路径移除配置的 **qB 下载根目录**，将所得相对路径拼到 **Stash 媒体库根目录**，再通知 Stash 扫描。只有两个服务已通过不同路径前缀访问同一份内容时才应使用。

文件交付先把同一相对路径拼到 **Moji 下载根目录** 得到源路径，再复制、移动或链接到 **Moji 媒体库根目录** 下的对应位置，最后拼到 **Stash 媒体库根目录** 生成扫描路径。目标已存在时会失败，Moji 不会覆盖。

`复制` 保留下载源文件；`移动` 成功后源文件不再保留，可能影响 qBittorrent 做种；`符号链接` 要求 Moji 主机始终能访问链接目标，通常不适合跨主机或仅容器内部可见的路径。
