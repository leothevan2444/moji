# 故障排除

先在 **主页** 检查服务是否就绪，再打开相关任务查看阶段错误和路径，最后到 **设置 > 日志** 查看后端详情。排查时先保留任务，其中存储的路径和上游标识是重要依据。

## 连接与搜索问题

- **已配置但未就绪**：字段完整，但最近探测失败或已过期。应从 Moji 服务端或容器验证 URL、凭据、TLS 信任和反向代理路径。
- **StashBox 无结果**：确认 Stash 至少配置一个 StashBox，在自动化设置中刷新列表，并检查 API key。Moji 按顺序尝试并返回第一个成功匹配的端点结果。
- **Jackett 无结果**：确认 Jackett 就绪、至少一个索引器已配置，并确认筛选条件未排除索引器。清除 tracker 筛选后用准确番号重试。
- **无可下载候选**：搜索结果没有可用磁力或 torrent 地址，或者规则未留下可用项。人工处理受阻任务，或调整选种规则。
- **重复被拒绝**：Moji 会阻止相同番号或种子的第二个任务，并检查 Stash 中是否已有番号。应使用已有任务或媒体库条目。

## 下载、入库与扫描问题

- **进度不更新**：点击 **同步进度**，再确认种子仍在当前配置的 qBittorrent 中，并且标识与任务一致。
- **内容路径不在 qB 根目录下**：qBittorrent 返回的路径不属于所配置 qB 下载根目录。应使用 qBittorrent 文件系统视角修正根目录，不能在该字段填写 Moji 主机路径。
- **文件交付失败**：检查 Moji 下载与媒体库根目录、权限、剩余空间，并确认目标不存在。使用符号链接时还要确认源路径持续可达。
- **扫描路径错误**：对比任务相对路径与 Stash 媒体库根目录。最终路径必须使用 Stash 文件系统视角，并位于已配置媒体库内。
- **Stash 扫描失败或停滞**：确认 Stash 就绪，并从任务查看扫描任务 ID 和错误。解决 Stash 后台任务或路径问题后再重试。

路径或权限问题的原因未改变时，不要反复重试。重试使用原任务记录和当前设置；两者都没有变化时，结果也不会变化。
